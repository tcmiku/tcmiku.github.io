253.如图所示，通过BGP EVPN动态建立VXLAN隧道的场景中，VTEP1发 送给VTEP2的BGP EVPN Type2路由所携带的RT值为以下哪项?

A.10: 1

B.200:1

C.20: 1

D.100:1

答案：A

解析：

在BGP EVPN动态建立VXLAN隧道的场景中，Type2路由携带的RT值遵循以下逻辑：

RT（Route Target）作用：

  • 导出（export）：路由发送时携带的RT值，标识此路由的归属或应用场景
  • 导入（import）：接收方通过匹配RT值决定是否接收路由
  • 为确保VTEP间路由互通，双方的导出RT需被对方的导入RT匹配

关键点：

  • Type2路由携带发送方（VTEP1）的导出RT值10:1，接收方（VTEP2）通过导入此RT获知路由。
  • RT配置需确保双方导出-导入对称（如VTEP2应配置import-rt 10:1，VTEP1配置import-rt 20:1）

254.在VXLAN实现中，VNI 与BD的映射关系是以下哪一项?

A.N:M

B.1:N

C.N:1

D.1:1

答案：D

解析：

在VXLAN的实现中，VNI（VXLAN Network Identifier）与BD（Bridge Domain）之间的映射关系是1:1，具体分析如下：

VNI的作用：

  • VNI是VXLAN段（覆盖网络）的唯一标识符（24位），类似传统VLAN的VLAN ID，但范围更大（0~16,777,215），用于区分不同的二层逻辑网络。

BD的作用：

  • 桥接域（BD）是二层广播域的抽象，用于组织同一二层网络内的终端设备，实现流量隔离。每个BD独立维护MAC地址表，并对应一个VXLAN段。

映射关系：

  • 1:1对应：每个BD必须绑定唯一的VNI，而每个VNI也只能映射到一个BD。

    例如，配置bridge-domain 10后指定vni 5000，即BD10与VNI 5000形成一对一关系。

  • 逻辑隔离需求：若多个BD共享同一VNI，会导致流量跨二层域泄露；若一个BD对应多个VNI，则会破坏VXLAN的标识逻辑。

总结：VNI与BD的严格一对一映射是VXLAN架构的基础，确保覆盖网络与底层二层广播域的一一对应关系，保障流量隔离与网络可靠性

255.华为SD -WAN解决方案使用以下哪项隧道技术建立数据通道?

A.GRE Over IPsec

B.VXLAN

C.MPLS

D.IPSec

答案：A

解析：

GRE Over IPsec的核心优势：

  • GRE（Generic Routing Encapsulation）：提供灵活的隧道封装能力，支持跨异构网络的多协议通信。
  • IPsec叠加GRE：
    1. 流程：数据先进行GRE封装（构建隧道），再经IPsec加密（保障安全）。
    2. 特点：
      • 兼容性高，适应复杂网络环境（如NAT穿透）。
      • 加密封装分离，便于实现流量优化（如智能选路、QoS策略）。
  • 华为SD-WAN的实现：
    默认使用GRE Over IPsec作为Overlay隧道技术，既满足企业对广域网连接的安全性要求（IPsec加密），又支持灵活的流量调度与负载均衡（GRE隧道）

260.在VXLAN网络中，用于实现VXLAN网络内跨子网通信以及VXLAN网络访问外部网络(非VXLAN网络)的是以下哪项?

A.VXLAN L2 Gateway

B.VXLAN L3 Gateway

C.NVE接口

D.VLANIF接口

答案：B

解析：

  1. VXLAN L3 Gateway的作用
    • 跨子网通信
      • VXLAN L3网关为VXLAN网络提供三层路由功能。
      • 当不同VXLAN子网（VNI对应不同网段）或非VXLAN网络之间需要通信时，L3网关负责IP报文的路由和转发。
      • 示例：VXLAN域内的VM1（网段10.1.1.0/24）需与VM2（网段10.1.2.0/24）通信，L3网关通过路由表实现子网间互通。
    • 访问外部网络
      • L3网关作为VXLAN网络的边界出口，通过配置默认路由或静态路由，将流量导向外部网络（如传统IP网络或Internet）。
      • 示例：VM1（10.1.1.2）访问公网服务器（203.0.113.5），L3网关完成源目的地址转换（如NAT）并转发至外部接口。
  • VXLAN L3 Gateway（如华为的CloudEngine交换机）是跨子网通信及对外互通的核心组件。
  • L2网关、NVE接口均不参与三层路由，仅解决二层扩展问题。而VLANIF接口在VXLAN架构中不再承担主路由角色

272.OSPFv2是运行在IPv4网络的IGP，0SPFv3是 运行在IPv6网络的IGP。OSPFv3与0SPFv2的报文类型相同，包括: He11o报文、DD报文、LSR报文、LSU报文和LSAck报文。以下关于0SPFv3报文的描述，正确的是哪一项?

A.OSPFv3使用报文头部的认证字段完成报文认证

B.OSPFv3使用IPv6组播地址FF02: : 5和FF02: :6发送0SPFv3报文

C.OSPFv3的He11o报文携带了路由器接口的IPv6地址前缀列表

D.OSPFv3使用链路本地地址作为发送报文的源地址，报文可以被转发到始发链路范围之外

答案：B

解析：

  1. 选项A
    • 描述：OSPFv3使用报文头部的认证字段完成报文认证。
    • 错误原因：OSPFv3的认证机制与OSPFv2不同，不再依赖报文头部字段，而是通过IPv6的IPSec实现认证与安全性。
    • 结论：错误。
  2. 选项B
    • 描述：OSPFv3使用IPv6组播地址FF02::5和FF02::6发送报文。
    • 正确原因：
      • FF02::5：对应所有OSPFv3路由器（AllSPFRouters），用于发送通用OSPFv3报文（如Hello、LSU）。
      • FF02::6：对应指定路由器（AllDRouters），用于DR/BDR选举相关的通信。
    • 验证：与OSPFv2组播地址（224.0.0.5/224.0.0.6）逻辑一致，IPv6组播地址符合协议要求。
    • 结论：正确。
  3. 选项C
    • 描述：OSPFv3的Hello报文携带接口的IPv6地址前缀列表。
    • 错误原因：
      • OSPFv3的Hello报文不再携带IP地址或前缀信息，而是通过Link-LSA（链路状态通告）传递IPv6前缀和网络掩码。
      • Hello报文仅包含接口ID、Router ID、邻居列表等二层链路信息。
    • 结论：错误。
  4. 选项D
    • 描述：OSPFv3使用链路本地地址作为源地址，且报文可被转发到其他链路。
    • 错误原因：
      • 链路本地地址仅在本地链路有效（即同一广播域），无法跨链路路由。
      • OSPFv3报文虽以链路本地地址为源，但其通信范围始终限制在本地链路内，无法被转发到其他网络。
    • 结论：错误。

280.OSPFv2是运行在IPv4网络的IGP。OSPFv3是运行在IPv6网络的IGP。OSPFv3的基本 工作机制与OSPFv2相同，例如DR和BDR的选举。如图，某工程师利用3台路由器进行IPFv6业务测试，他想要通过运行OSPFv3实现IPv6网络的互联互通。以下关于该网络DR选举的描述，正确的是哪一项?

A.网络收敛后R1断电重启，网络再次收敛时，R3为DR

B.网络收敛后，该广播链路新增一台设备R4, R4的优先级为150.网络再次收敛时，R4为DR

C.网络收敛后，R1为DR

D.网络收敛后，工程师修改R1的优先级为100. R2的优先级为10，R3保持缺省值。网络再

答案：A

解析：

OSPFv3的选举规则与OSPFv2相同，由以下两个优先级因素决定：

  • a. 优先级（Priority）
    • 接口的优先级范围是 0-255，默认值为1。
    • 0优先级表示该接口不参与DR/BDR选举。
    • 若某路由器的优先级最高，则成为DR；次高者成为BDR。
  • b. Router ID
    • 若优先级相同，则比较Router ID，数值较大的胜出。
    • 必须手动配置Router ID（OSPFv3中IPv6地址不直接作为Router ID）

A.DHCP报文中的0ptions字段采用ILV结构，可以携带如NTP服务 器地址、AC地址和日志服务器地址等参数信息

B.DHCP支持给指定的终端分配固定的IP地址，这种分配机制被叫做DHCP静态分配

C.若只需要给DHCP客户端下发网关和DNS服务器的信息，则DHCP服务器发送给客户端的DHCP Offer报文将不会携带0ptions字段

D.当DHCP客户端与DHCP服务器处在不同的三层网络时，需要配置DHCP中继实现客户端与服务器之间的报文转发

答案: C

解析：

DHCP关键参数（如子网掩码、租期、网关、DNS）均通过Options字段传递。即使仅下发网关（Option 3）和DNS（Option 6），仍需通过Options字段。若Options缺失，客户端无法获取必要参数。此描述错误。

285.如图所示，在虚拟化园区网络中部署业务随行，其中PC1属于Sales安全组， PC2属于R&D安全组，PC3属于Market安全组。 以下关于该场景的描述中，错误的是哪项?

286.华为SD-WAN解决方案中，当CPE位 于NAT设备后的私网时，特别是两个站点的CPE同时位于NAT设备后的私网时，CPE之 间需要使用NAT穿越技术。华为SD-WAN解决方案中使用以下哪一项技术帮助CPE之间实现NAT穿越?

B.NAT server

C.IPsec VPN

D.STUN

通过STUN（Session Traversal Utilities for NAT）协议，CPE可通过公网STUN服务器探测自身的NAT映射地址与端口，并将其上报给SD-WAN控制器。控制器协调双端建立连接时，基于STUN探测到的公网信息直接通信。是华为SD-WAN解决对称NAT穿透的标准技术

289.某企业的广域承载网中存在各种不同的业务，若想在网络出现拥塞时保证协议报文的转发和处理，则应该将协议报文的优先级设置为以下哪一项?

A.BE

B.CS6/CS7

C.EF

D.AF4

答案：B

解析：

Class Selector 6/7（DSCP值48/56）专用于网络控制流量（如路由协议、管理协议）。CS6通常用于普通控制平面流量（如OSPF、BGP），CS7保留用于更关键的网络信令（如STP、VRRP）。此类流量在网络设备中被优先处理

协议报文的优先级应设置为CS6/CS7，以确保网络拥塞时控制流量优先转发，维持网络稳定性和可靠性

290.相比较于0SPFv2报文头部，以下哪项是OSPFv3报文头部增加的字段?

A.Instance ID

B.Link ID

C.IPv6 Auth Type

D.IPv6 Router ID

答案 ： A

解析：
OSPFv3 针对 IPv6 进行了协议优化，其报文头部相较于 OSPFv2 增加了 Instance ID 字段，用于在同一物理链路上区分多个独立的 OSPFv3 实例（例如不同虚拟网络或租户场景）。以下是各选项的详细分析：

  1. Instance ID（选项A）
    • 作用：8 位字段，允许多个 OSPFv3 实例共享同一链路，通过不同 Instance ID 隔离流量。
    • OSPFv2 对比：OSPFv2 无此字段，同一链路上仅支持单一实例

Instance ID 是 OSPFv3 报文头部新增的核心字段，支持多实例场景，其他选项均不涉及报文头部的结构变更

291.以下关于Sham link的描述，错误的是哪一项?

A.为了使VPN流量通过IPLS骨干网转发，Sham 1ink的路由开销应小于backdoor链路的OSPF路由开销

B.Sham link被看成是两个VPN实例之间的链路，每个VPN实例中必须有一个Sham 1ink的端点地址，它必须是PE路由器上VPN地址空间中的一个有32 bi t掩码的Loopback接口地址

C.Sham 1ink的端点地址被BGP作为VPNv4地址发布。经过了Sham link的路由也可再以VPNv 4路由的形式被引入到BGP

D.Sham link可以在任何区域用手工配置，并且本端VPN实例中必须有到Sham link目的地址(对端PE的Sham link端点地址)的路由

答案：D

解析：

  • 区域限制：Sham link必须配置在同一个OSPF区域内（一般为Backbone Area 0）。如果两个PE的VRF实例属于不同的OSPF区域（如Area 1和Area 2），无法直接建立Sham link，因为OSPF要求虚拟链路必须位于骨干区域。
  • 路由可达性：本端VPN实例需要到对端Sham link端点地址的路由（例如通过BGP或静态路由），这是正确的要求。但选项D的“任何区域”配置与OSPF区域规则冲突，因此整体描述错误。

总结
选项D的前半部分“可以在任何区域”是错误的关键，Sham link对区域配置有严格限制；后半部分路由可达性要求是合理的。因此正确答案为D。

293.OSPFv2是运行在IPv4网络的IGP，OSPFv3是运行在IPv6网络的IGP。OSPFv3与OSPFv2相比，具有类似的LSA类型，但是功能略有区别。以下哪种LSA不属于OSPFv3的LSA类型?

A.ASBR- Summary-LSA

B.AS-External-LSA

C.Network-LSA

D.Router-LSA

答案： A

解析：

在OSPFv3中，ASBR-Summary-LSA（类型4） 的名称已被更改为 Inter-Area-Router-LSA，尽管其功能仍用于通告ASBR的位置，但名称不再沿用OSPFv2的术语。因此，严格来说，“ASBR-Summary-LSA”这一名称不属于OSPFv3的LSA类型

294.通过iMaster NCE-Campus部署的虚拟化园区网络场景中，以下关于“站点”的描述，错误的是哪一项?

D.一个站点即租户管理员创建的一个租户网络

295.除了虚连接之外，OSPFv3的Hello报文源IPv6地址是以下哪种类型的IPv6地址？

A.唯一本地地址

B.全球单播地址

C.IPv6任播地址

D.链路本地地址

答案：D

解析：
OSPFv3 的 Hello 报文在非虚连接场景下，其源 IPv6 地址为 链路本地地址（Link-Local Address，fe80::/10），这是由协议设计决定的。以下是关键点分析：

  1. 链路本地地址的核心作用
    OSPFv3 要求相邻路由器通过链路本地地址直接通信，确保协议报文仅在本地链路传输，无需依赖全局地址。Hello 报文作为邻居发现与维护的关键消息，源地址必须为链路本地地址，以保证邻居关系建立的可靠性。

296.在割接的测试阶段中，哪一项不是表明己经完成测试的标准？

A.查看网络运行正常为标准

B.测试网络业务正常为标准

C.网络承载的上层应用业务测试正常为标准

D.IP设备的配置查看结果正常为标准

答案：D

解析：

完成测试的核心在于实际运行状态的验证，而非配置的静态检查。选项D仅关注配置正确性，无法全面反映割接后的网络健康状态，因此不是完成测试的标准

297.以下关于MPLS网络中配置静态LSP的描述，错误的是哪一项？

A.当某一台LSR为Transit LSR时，需要同时配置In Label和Outlabel，In Label范围为161023，Out Labe1范围为161048575

B.当某一台LSR为Egress LSR时，仅需配置In Label,范围为16~1023

C.当某一台LSR为Transit LSR时，需要同时配置In Label和Outlabel,范围均为16~1023

D.当某一台LSR为Ingress LSR时，只需配置Out Label,范围为16~1048575

答案：C

解析：

选项C错误在于限定了Transit LSR的Out Label范围为161023，而实际上其出标签范围应为161048575。其他选项均符合静态LSP的配置规则

299.在VXLAN网络中，以下哪一项标识类似传统网络的VLAN ID,用于区分VXLAN段？

VNI直接承载在VXLAN报文中，用于标识不同的虚拟网络段，与传统VLAN ID的作用完全一致

300.当IS—IS网络中有多条冗余链路时，可能会出现多条等价路由。以下关于IS—IS网络内等价路由的描述，错误的是哪一项？

A.若配置负载分担，则流量会被均匀地分配到每条链路上

B.当组网中存在的等价路由数量大于通过命令配置的数量，且这些路由优先级相同时，优选下一跳设备System ID大的路由进行负载分担

C.针对等价路由中的每一条路由，可以指定优先级，优先级高的路由会被优选，其余的作为备份路由

D.配置等价路由优先级后，IS—IS设备在转发到达目的网段的流量时，将不采用负载分担方式，而是将流量转发到优先级最高的下一跳

答案: B

解析：

当等价路由数量超过配置的最大负载分担数时，需通过规则筛选部分路由。华为设备优先选择下一跳System ID较小的路由加入负载分担，而非System ID较大的
